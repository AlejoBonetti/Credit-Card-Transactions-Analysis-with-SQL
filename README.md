# Credit Card Customer Analytics with SQL


## Dataset
The original dataset contains approximately 20 millions transactions. Due to MySQL import limitations on my pc, this project uses a random sample of 1 million transactions while preserving the original table structure and relationships.

- **Source**: Kaggle - *Synthetic IBM credit card dataset*
- 20 million transactions. 
- 3 original tables.

*Link:* https://www.kaggle.com/datasets/ealtman2019/credit-card-transactions/

## Database Structure

| Table    | Description     |
| ------------- | ------------- |
| Transactions | Every credit card transaction |
| Users | Customer demographic information |
| Cards | Credit card information for each customer |
| users_clean / cards_clean / trans_clean | Cleaned versions used for analysis |


## Project Objectives

- Analyse transaction trends through time.
- Identify high-value customers.
- Explore the relationship between income and spending.
- Compare spending behaviour across card types.
- Analyse spending by credit limit groups.
- Detect unusual customer spending using statistical methods.
- Perform cohort analysis to study customer spending after acquisition.

## Data Cleaning

Before the analysis, the dataset was cleaned to improve consistency and simplify queries.

- Currency symbols were removed.
- Monetary columns were converted to numeric.
- Clean tables were created and used throughout the project.
- Standardized column names.
- Table relationships were verified before the analysis.

## Business Questions

1. How do transactions change through time? Revenue by month, number of transactions and monthly growth.
2. Which clients spend the most throughout their lifetime? Top 10 clients and their contribution to total revenue.
3. How much revenue is generated by the top spending customers?
4. How does average yearly spending vary across income groups?
5. How do spending patterns differ across card types?
6. How does transaction size vary across different credit limit groups?
7. Which clients have unusual spending?
8. How does customer spending evolve after acquisition? (Cohort Analysis)

## Key Findings

- Revenue became progressively more stable as the number of transactions increased over time.
- The top 20% of customers account for 50.26% of total revenue.
- Average yearly spending increases consistently across income quartiles, with customers in the highest income quartile spending nearly twice as much as those in the lowest income quartile.
- Average transaction size increases across credit limit quartiles, although the growth is modest relative to the increase in available credit.
- Only 1.5% of customers exhibit unusual spending relative to peers with similar income, as indicated by absolute Z-scores greater than 3.
